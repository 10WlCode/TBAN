looks like that was generated with react-native init and not create-react-native-app, because it doesn't have an "expo" section. we should provide a better error message in this case.

app.json in a project created with create-react-native-app looks like this:

{
  "expo": {
    "sdkVersion": "18.0.0"
  }
}

(copied and pasted directly from a project that I just created)

cc @wilzh40 it seems that xdl validation can be improved in this case.

neelotpalnag on 29 Aug 2017

If your app.json looks like :
{ "name": "testapp", "displayName": "testapp" }

you can fix the issue by adding the expo SDK version to the above JSON, as :

{ "name": "testapp", "displayName": "testapp", "expo": { "sdkVersion": "18.0.0" } }

(You may have a different SDK version. So check for that)
