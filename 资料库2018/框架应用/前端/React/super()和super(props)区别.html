react项目为什么要super(props)？

react项目中子类继承component为什么要super(props)，传props这个参数的目的是什么？我知道这个super现在相当于Component.prototype.constructor.call(this,props)，但具体这么做的原因是为什么？为什么要往component的constructor中传props？

这是ES6语法写React组件的一种方式。

因为在React中，默认的数据流流向是单方向的，只能从父组件流向子组件，早在ES5写法中，父组件向子组件传值时，是通过props属性传递的。

所以，在ES6写法中，通过super(props)实现子组件对父组件传值的接收。这里是必须的和固定的写法。

在其他面向对象的编程语言中，也是有super这个关键字的，super是父类、子类继承的一种方式。

并不简单的是你所理解的，super(props) 之后，可以通过 this.props 访问。

这里 super(props) 之后，能在类里通过 this 访问是因为在 React.Component 里面把传入的 props 绑定到了 this 上去，于是在 super 之后就可以用 this 访问了。

https://github.com/facebook/react/blob/master/src/isomorphic/modern/class/ReactBaseClasses.js#L24

function ReactComponent(props, context, updater) {
  this.props = props;
  this.context = context;
  this.refs = emptyObject;
  // We initialize the default updater but the real one gets injected by the
  // renderer.
  this.updater = updater || ReactNoopUpdateQueue;
}

而绑定 props 则是为了之后的组件的实现，没有 props 就什么都做不了了。
